Scenario: Site documents

Document information

Information about the document itself that provides useful context when evaluating the document's contents.

Site documents describe monitoring points (geographical places where measurements were observed) and sites (which represent a group of related monitoring points). Site documents should be transmitted when a new monitoring point is commissioned or an existing monitoring point changes.

You can encode as many sites and monitoring points into one document as you like.

A revised site or monitoring point completely replaces the previous definition – there are no partial updates, only complete replacements.

Ratings documents

Ratings documents describe the rating table which explains how volumetric data may be derived from water level.

They consist of:

  • Gauging observations - raw observered measurements which are used to establish a rating table (sometimes referred to as a ratings curve).
  • A rating table - a set of value pairs that relate discharge to level. This table is derived from the gauging observations.

It is not expected that all data providers will have access to ratings data.

t should be noted that Ratings documents use WaterML2.0 Part 2 - Ratings and Gaugings and Sections, so the XML namespace and schema is different to other files in the Austrailian Profile of WaterML2 which only use WaterML2.0 Part 1.

Time series documents

Time series documents describe time series data (sequences of measurements) and metadata about:

  • How to interpret the sequence of data points (time series metadata);
  • How the data points were produced (procedural metadata); and
  • How to interpret each individual data point (measurement metadata).

One long time series can be transmitted in sections. This allows ongoing time series measurements to be transmitted in hourly or daily chunks instead of waiting for the measurements to finish (which could be indefinite) and transmitting years of measurements in one hit.

Time series data that is improved (for example, a quality control process is run over the data improving the dataset) forms a new time series and must have a new identifier. So two time series exist, the original low quality time series and the later high quality time series.

Time series data can be revised (for example, to correct errors) – the revised data will completely replace the original data over the time range specified.

Each time series document must contain only one time series, although that time series can be split across many sections.
